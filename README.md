STDV - standart deviation (стандартное отклонение, мера разброса чисел вокруг их среднего)
Надо взять окно из некольких сосдених дальностей и посчитать их среднее u
o = sqrt(avg(Lr-u)^2)
Закономерность: если в окне дальности гладкие, то `o` маленькое (сплошная стена - соседние лучи меняются плавно), если дальности скачут, то `o` большое (лучи проходят сквозь стекло и попадают куда кто, еще надо учестьдропауты)

В исследовании все делалось на 3d лидаре, но у меня 2д, скозльзящее окно бежит вокруг азимута

PCoG - Point Cloud of Glass - облако точек стекла
PCoO - Point CLoud of Obstacle облако точек объектов

Калибровка: Авторы сначала самостоятельно сидели и размечали данные на две группы PCoG и PCoO, считают STDV в каждой `oPCoG` и `oPCoO`

Алгоритм 1:
 - скользящее окно размером 11 точек (5 точек до текущей точки и 5 точек после)
 - для каждого окна надо высчитать STDV дальностей
 - если STDV > `Threshold`, то точки надо пометить как кондидаты в PCoG
 - иначе пометить как кондидаты в PCoO

Алгоритм 1 представляет собой грубый поиск "стекла" по окнам

Алгоритм 2:
 - Начало стекла это увеичение дальности и одновременное уменьшение интенсивности
 - Конец стекла это увеличение интенсивности и уменьшение дальности

Алгоритм 2 нужен для того, чтобы проверить, а действительно ли это стекла. Тут проверяется его границы

Алгоритм 3:
 - `D1` - дальность на одном краю стекла, `D2` - на другом 
 - `N` - число лучей в этом окне
 - `Uc` - шаг. `Uc = (D2 - D1) / N`
 - Нужно пройти по `N` лучам и прибавить `Uc` с накоплением (k-й луч = D1 + k * Uc)
